Swarm Intelligence Routing
Every node independently scores its neighbors using a reinforcement formula: value = route_score − load_penalty. Reliable paths are rewarded; congested ones are avoided automatically.
Core Algorithm
def choose_next(self, neighbors):
scores = {}
for n in neighbors:
score = self.route_scores[n]
penalty = n.load * 0.5
scores[n] = score - penalty
return max(scores, key=scores.get)

Faculty Q&A

Expected questions

Why bees? +
Bees use swarm intelligence — no central control, each bee makes local decisions that result in globally optimal behavior. Our nodes do the same: each one independently scores paths, penalizes load, and routes packets without any global coordinator.
How is this different from Dijkstra? +
Dijkstra finds the shortest static path. Our algorithm adapts dynamically based on real-time load and historical delivery success. It's closer to reinforcement learning at the network level — paths improve over time based on outcomes.
What is the Queen's role? +
The Queen is the least-loaded active node at any time. She acts as coordinator for priority traffic. If she fails, re-election is immediate and automatic — the next least-loaded node is crowned. The network never stops working.
What is the routing algorithm exactly? +
Score-based greedy routing with load penalty: value = route_score − load_penalty. Each node picks the neighbor with the highest value. When a packet is delivered successfully, all nodes on the path get their route scores increased — reinforcement learning style.
Why SQLite and not a real database? +
SQLite is lightweight, needs zero installation, and is built into Python's standard library — perfect for a simulation and demo environment. For production, the same schema works directly with PostgreSQL.
What happens if all nodes fail? +
The election function returns None, and the Flask API returns a graceful error response with a clear message. The system handles total network collapse without crashing — it waits for at least one node to recover.
